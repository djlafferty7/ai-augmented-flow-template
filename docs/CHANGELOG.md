- **[conventional-changelog](https://github.com/conventional-changelog/conventional-changelog)**: `npx conventional-changelog -p angular -i docs/CHANGELOG.md -s`
- **GitHub Releases**: Create releases from tags; GitHub auto-generates release notes from PR titles.

## Commit Message Quality

Since the changelog is derived from commits, write meaningful commit messages:

- **Format:** `<type>(<scope>): <description>` (see [CONTRIBUTING.md](CONTRIBUTING.md))
- **Body:** Explain *why*, not just *what*. Reference GitHub Issue numbers.
- **Breaking changes:** Use `BREAKING CHANGE:` footer or `!` after type.
